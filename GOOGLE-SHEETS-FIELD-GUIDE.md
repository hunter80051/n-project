# Google Sheets 遊戲配置欄位完整說明

本文件由 `tools/game_data_schema.json` 產生，說明 Google Sheet 八張資料分頁的欄位、填寫規範與目前程式支援狀態。

實作狀態定義：

- **已實作**：目前遊戲會讀取並使用此欄位。
- **部分實作**：欄位有被讀取，但只有部分值或行為已接入遊戲。
- **尚未實作**：欄位目前只作為預留或文件資訊，修改不會改變遊戲行為。

所有 ID 關聯都依字串比對，不依賴資料列順序；但角色顯示順序、地下城輪替順序與卷軸顯示順序仍會受到資料列順序影響。

## 1. `balance`

- CSV：`data/balance.csv`
- 用途：全域模擬、生成與成長參數。
- 資料列識別：`key`

### 欄位

- `key`
  - 功能：參數的唯一名稱，程式以 data.balance[key] 讀取。
  - 填寫規範：不可空白或重複；使用英文字母開頭的英數 camelCase。
  - 實作狀態：**已實作**。
- `value`
  - 功能：參數值，依 type 轉為 number、boolean 或 string。
  - 填寫規範：必須符合 type；number 不可為 NaN。
  - 實作狀態：**已實作**。
- `type`
  - 功能：控制 value 的載入型別。
  - 填寫規範：只可填 number、boolean、string。
  - 實作狀態：**已實作**。
- `description`
  - 功能：供配置維護者閱讀的參數說明。
  - 填寫規範：請清楚描述單位與影響範圍。
  - 實作狀態：**尚未實作**。
  - 補充：不會被遊戲執行期讀取。

## 2. `characters`

- CSV：`data/characters.csv`
- 用途：固定小隊角色、戰鬥定位、外觀索引與基礎能力。
- 資料列識別：`characterId`

### 欄位

- `characterId`
  - 功能：角色唯一 ID，供技能選擇、UI 與執行期狀態引用。
  - 填寫規範：不可空白或重複；格式為小寫英數及底線。
  - 實作狀態：**已實作**。
- `name`
  - 功能：角色顯示名稱。
  - 填寫規範：不可空白。
  - 實作狀態：**已實作**。
- `role`
  - 功能：角色職責顯示；Healer 會優先成為治療卷軸施法者。
  - 填寫規範：目前建議 Tank、Healer、Caster、Striker。
  - 實作狀態：**部分實作**。
  - 補充：只有 Healer 具有額外程式行為，其餘主要供顯示。
- `combatStyle`
  - 功能：控制近戰／遠程站位與投射物行為。
  - 填寫規範：只可填 melee 或 ranged。
  - 實作狀態：**已實作**。
- `attackType`
  - 功能：控制遠程投射物與攻擊視覺類型。
  - 填寫規範：只可填 physical 或 magic。
  - 實作狀態：**已實作**。
- `color`
  - 功能：角色 UI 與 Canvas 主色。
  - 填寫規範：使用 #RRGGBB。
  - 實作狀態：**已實作**。
- `spriteId`
  - 功能：選擇角色程式繪圖外觀。
  - 填寫規範：目前使用 sprite_dinu、sprite_hoya、sprite_lynn、sprite_bob。
  - 實作狀態：**部分實作**。
  - 補充：新 spriteId 不會自動產生專屬造型。
- `maxHp`
  - 功能：角色初始最大生命值。
  - 填寫規範：大於 0。
  - 實作狀態：**已實作**。
- `maxSp`
  - 功能：角色初始最大 SP。
  - 填寫規範：不得小於 0。
  - 實作狀態：**已實作**。
- `attack`
  - 功能：角色基礎攻擊力。
  - 填寫規範：不得小於 0。
  - 實作狀態：**已實作**。
- `defense`
  - 功能：角色基礎防禦力。
  - 填寫規範：不得小於 0。
  - 實作狀態：**已實作**。
- `speed`
  - 功能：角色每秒移動格數。
  - 填寫規範：必須大於 0。
  - 實作狀態：**已實作**。
- `attackRange`
  - 功能：角色可攻擊距離，單位為地圖格。
  - 填寫規範：必須大於 0；遠程通常大於近戰。
  - 實作狀態：**已實作**。
- `attackCooldownMs`
  - 功能：普通攻擊冷卻毫秒數。
  - 填寫規範：必須大於 0。
  - 實作狀態：**已實作**。
- `basicSkillId`
  - 功能：角色基本技能 ID。
  - 填寫規範：必須存在於 skills.skillId。
  - 實作狀態：**已實作**。
- `skillPoolId`
  - 功能：升級時抽選技能的技能池 ID。
  - 填寫規範：skills 中至少要有一列使用相同 skillPoolId。
  - 實作狀態：**已實作**。

## 3. `skills`

- CSV：`data/skills.csv`
- 用途：角色基本技能與升級被動技能。
- 資料列識別：`skillId`

### 欄位

- `skillId`
  - 功能：技能唯一 ID。
  - 填寫規範：不可空白或重複；格式為小寫英數及底線。
  - 實作狀態：**已實作**。
- `skillPoolId`
  - 功能：技能所屬升級池。
  - 填寫規範：須與角色 skillPoolId 對應。
  - 實作狀態：**已實作**。
- `name`
  - 功能：技能顯示名稱。
  - 填寫規範：不可空白。
  - 實作狀態：**已實作**。
- `description`
  - 功能：技能選擇介面的說明文字。
  - 填寫規範：請與實際效果一致。
  - 實作狀態：**已實作**。
- `effectType`
  - 功能：指定基本攻擊、治療或被動能力的程式效果。
  - 填寫規範：只可使用目前列出的類型；新增類型需修改 JavaScript。
  - 實作狀態：**部分實作**。
  - 補充：guardStrike 目前只使用傷害倍率，說明中的暫時防禦尚未實作。
- `targetType`
  - 功能：區分個人與全隊被動，並描述基本技能目標。
  - 填寫規範：只可填 enemy、ally、self、party。
  - 實作狀態：**已實作**。
- `power`
  - 功能：傷害／治療倍率或被動加成值。
  - 填寫規範：不得小於 0；百分比以小數表示。
  - 實作狀態：**已實作**。
- `spCost`
  - 功能：基本技能消耗 SP。
  - 填寫規範：不得小於 0。
  - 實作狀態：**已實作**。
- `cooldownMs`
  - 功能：技能冷卻毫秒數。
  - 填寫規範：不得小於 0。
  - 實作狀態：**部分實作**。
  - 補充：治療基本技能會使用；一般攻擊通常由角色 attackCooldownMs 決定。
- `requiredLevel`
  - 功能：升級選項可出現的最低角色等級。
  - 填寫規範：大於等於 1 的整數。
  - 實作狀態：**已實作**。
- `iconId`
  - 功能：預留技能圖示索引。
  - 填寫規範：使用小寫英數及底線。
  - 實作狀態：**尚未實作**。
  - 補充：目前技能 UI 不讀取 iconId。

## 6. `loot_tables`

- CSV：`data/loot_tables.csv`
- 用途：敵人掉落候選；每一列目前獨立依 chance 投擲。
- 資料列識別：`lootTableId + dropType + dropId`

- `lootTableId`
  - 功能：將多個掉落候選分到同一掉落表。
  - 填寫規範：至少須被一名敵人的 lootTableId 引用。
  - 實作狀態：**已實作**。
- `dropType`
  - 功能：指定掉落內容為裝備或卷軸。
  - 填寫規範：只可填 item 或 scroll。
  - 實作狀態：**已實作**。
- `dropId`
  - 功能：掉落物 ID。
  - 填寫規範：依 dropType 指向 items.itemId 或 scrolls.scrollId。
  - 實作狀態：**已實作**。
- `weight`
  - 功能：預留權重抽選值。
  - 填寫規範：不得小於 0。
  - 實作狀態：**尚未實作**。
  - 補充：目前不做權重抽選。
- `chance`
  - 功能：該列獨立掉落機率。
  - 填寫規範：0 到 1；0.25 代表 25%。
  - 實作狀態：**已實作**。
- `minQuantity`
  - 功能：成功掉落時的最小數量。
  - 填寫規範：大於等於 1，且不可大於 maxQuantity。

- `bossEnemyId`
  - 功能：第 3 層 Boss ID。
  - 填寫規範：必須存在於 enemies，且 isBoss=true。
  - 實作狀態：**已實作**。
- `difficultyScale`
  - 功能：該地下城敵人 HP、攻擊與防禦倍率。
  - 填寫規範：必須大於 0。
  - 實作狀態：**已實作**。

## 共通防呆規則

- 不可修改、翻譯或重新排列第一列表頭。
- 不可使用公式；同步工具只接受固定值。
- 新增資料可以使用新 ID；刪除或改名既有穩定 ID 預設會阻擋。
- 百分比與機率使用 0–1 小數，例如 `0.25` 代表 25%。
- `enemyPool` 使用 `|` 分隔，不可使用逗號。
- 只有同步檢查全部通過時，才可覆寫 repository 內的 CSV。
- 未實作欄位仍必須保留，避免破壞既有 CSV schema。
